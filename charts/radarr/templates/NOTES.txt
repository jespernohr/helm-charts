1. Get the application URL by running these commands:
{{- if .Values.ingress.enabled }}
{{- range $host := .Values.ingress.hosts }}
  {{- range .paths }}
  http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}{{ .path }}
  {{- end }}
{{- end }}
{{- else if contains "NodePort" .Values.service.type }}
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "radarr.fullname" . }})
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo http://$NODE_IP:$NODE_PORT
{{- else if contains "LoadBalancer" .Values.service.type }}
     NOTE: It may take a few minutes for the LoadBalancer IP to be available.
           You can watch the status of by running 'kubectl get --namespace {{ .Release.Namespace }} svc -w {{ include "radarr.fullname" . }}'
  export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "radarr.fullname" . }} --template "{{"{{ range (index .status.loadBalancer.ingress 0) }}{{.}}{{ end }}"}}")
  echo http://$SERVICE_IP:{{ .Values.service.port }}
{{- else if contains "ClusterIP" .Values.service.type }}
  export POD_NAME=$(kubectl get pods --namespace {{ .Release.Namespace }} -l "{{ include "radarr.selectorLabels" . }}" -o jsonpath="{.items[0].metadata.name}")
  export CONTAINER_PORT=$(kubectl get pod --namespace {{ .Release.Namespace }} $POD_NAME -o jsonpath="{.spec.containers[0].ports[0].containerPort}")
  echo "Visit http://127.0.0.1:7878 to use your application"
  kubectl --namespace {{ .Release.Namespace }} port-forward $POD_NAME 7878:$CONTAINER_PORT
{{- end }}

2. Radarr Configuration:
   - Web UI will be available at port 7878
   - Default credentials: No authentication required initially (configure in Settings → General)
   - Configure your media paths in Settings → Media Management
   - Add indexers in Settings → Indexers
   - Add download clients in Settings → Download Clients

{{- if .Values.persistence.enabled }}
3. Your Radarr configuration is persisted in the config volume.
{{- else }}
3. WARNING: Persistence is disabled! Your configuration will be lost when the pod is restarted.
{{- end }}

{{- if .Values.persistence.media.enabled }}
4. Media storage is configured and mounted at /movies in the container.
{{- else }}
4. Media storage is not configured. Consider enabling persistence.media for your movie library.
{{- end }}

{{- if .Values.persistence.downloads.enabled }}
5. Downloads storage is configured and mounted at /downloads in the container.
{{- else }}
5. Downloads storage is not configured. Consider enabling persistence.downloads for your download client.
{{- end }}

6. Metrics are enabled and can be scraped from the metrics service.

   ServiceMonitor has been created for Prometheus Operator.
{{- end }}
{{- end }}
